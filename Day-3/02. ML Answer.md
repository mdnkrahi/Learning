# 1. What is Data Drift?

### Answer

Data drift occurs when the statistical properties or distribution of the input data change over time compared to the data used for training.

### Example

A fraud detection model is trained on data where most transactions are from desktops. Over time, the majority of transactions come from mobile devices. The input data distribution has changed, which is data drift.

### Why is it Important?

* Model receives unfamiliar data
* Prediction quality may decrease
* Retraining may be needed

### Interview Answer

> Data drift refers to changes in the input data distribution over time. Since the model encounters data that differs from its training data, performance can degrade. Monitoring data drift helps identify when retraining is required.

# 2. What is Model Drift?

### Answer

Model drift, also called concept drift, occurs when the relationship between input features and the target variable changes over time.

### Example

A loan approval model was trained before an economic downturn. After market conditions change, customer behavior and default patterns also change. Even if the input data looks similar, the learned relationship is no longer accurate.

### Difference from Data Drift

* **Data drift:** Input data distribution changes.
* **Model (concept) drift:** The relationship between inputs and outputs changes.

### Interview Answer

> Model drift occurs when the underlying relationship between features and the target changes over time, causing prediction accuracy to decline. It is typically addressed through monitoring, retraining, and model updates.

# 3. Why is Feature Engineering Important?

### Answer

Feature engineering is the process of transforming raw data into meaningful input features that improve model performance.

### Common Techniques

* Handling missing values
* Encoding categorical variables
* Scaling and normalization
* Creating interaction features
* Date and time feature extraction
* Text vectorization
* Feature selection

### Benefits

* Improves model accuracy
* Reduces noise
* Speeds up training
* Enhances generalization

### Interview Answer

> Feature engineering converts raw data into informative features that help machine learning models learn patterns more effectively. High-quality features often have a greater impact on model performance than simply choosing a more complex algorithm.

# 4. Why Use MLflow?

### Answer

MLflow is an open-source MLOps platform that helps manage the end-to-end machine learning lifecycle.

### Key Capabilities

* Experiment tracking
* Parameter logging
* Metric logging
* Artifact management
* Model registry
* Model deployment support

### Benefits

* Reproducibility
* Collaboration
* Centralized experiment management
* Easier model comparison
* Lifecycle management

### Interview Answer

> MLflow is used to track experiments, manage model versions, store artifacts, and register models. It improves reproducibility, collaboration, and deployment across the machine learning lifecycle.

# 5. How Do You Monitor a Production ML Model?

### Answer

Monitoring a production ML model involves continuously tracking both system health and model quality to ensure reliable predictions over time.

### Infrastructure Metrics

* CPU and memory usage
* GPU utilization (if applicable)
* API latency
* Throughput
* Error rates
* Container and pod health

### Model Metrics

* Prediction accuracy (when ground truth is available)
* Precision, recall, F1-score
* Confidence score distribution
* Prediction distribution
* Data drift
* Model drift

### Operational Workflow

1. Collect logs and metrics.
2. Visualize dashboards using tools such as Prometheus and Grafana.
3. Set alert thresholds (e.g., high latency or increased error rate).
4. Detect data or model drift.
5. Retrain and validate the model if performance declines.
6. Deploy the updated model using the CI/CD pipeline.

### Interview Answer

> I monitor production ML models by tracking infrastructure metrics like CPU, memory, latency, and error rates, as well as model-specific metrics such as accuracy, prediction distributions, data drift, and model drift. Using tools like Prometheus, Grafana, and MLflow, I can detect issues early, trigger alerts, and retrain or redeploy models when necessary.


## Interview Tip

A strong answer should emphasize that **machine learning is a continuous lifecycle rather than a one-time model training task**. Production systems require monitoring, versioning, automation, and retraining to keep models reliable and effective.
